id: audit-trail
title: Audit Trail
---

Der Audit Trail hilft Ihnen dabei nachzuhalten, wer was innerhalb von qmBase gemacht hat.
Ein Audit Event wird immer dann gespeichert, eine Änderung innerhalb der Software vorgenommen wurde.
Dazu zählt beispielsweise:

- Ändern des Verantwortlichen
- Ändern von Custom Fields
- Ändern von Zugriffsberechtigungen
- und viele mehr...

Der Audit Trail kann im Admin Bereich aktiviert oder deaktiviert werden.

:::info
Ereignisse, die im Audit Trail gespeichert werden, werden laufend erweitert. Derzeit wird noch nicht für jede Änderung ein Audit Event gespeichert.
:::

In den Nachfolgenden Tabellen für einzelne Apps die Aktionen gekennzeichnet, die ein Audit Event auslösen.

## Zielmanagement

| Aktion                            |
| --------------------------------- |
| Ändern des Verantwortlichen       |
| Hinzufügen eines Kommentars       |
| Ändern des Status                 |
| Ändern von Schlagworten           |
| Ändern der Zugriffsberechtigungen |
| Ändern von Custom Fields          |
