id: feedback
title: Feedback
---

qmBase wird von uns so offen und transparent wie möglich weiterentwickelt. Innerhalb von qmBase können Sie uns jederzeit Verbesserungsvorschläge oder Fehler melden.
:::success Danke für Ihr Feedback
Ihr Feedback ist unser wichtigster Ausgangspunkt, um qmBase zu verbessern. Basierend auf Ihrem Feedback entwicklen wir unsere Roadmap und finden Fehler. Daher freuen wir uns **immer** über Ihre Rückmeldungen.
:::
Damit Sie eine gute Vorstellung davon haben, wie wir mit Ihrem Feedback umgehen hier die wichtigsten Schritte, die Ihr Vorschlag durchläuft.

1. Über den Button Verbesserung Vorschlagen schicken Sie uns Ihr Feedback. Hierbei handelt es sich entweder, um Verbesserungen oder Fehlermeldungen.
2. Sie erhalten sofort nachdem der Vorschlag bei uns eingangen ist direkte Rückmeldung und eine Ticketnummer, so dass ihr Vorschlag identifizierbar ist.
3. Wenn Sie einen Fehler übermitteln, probieren wir diesen zu reproduzieren. Ist das möglich, wird das Ticket mit dem Schlagwort <code>Confirmed</code> versehen. Wenn wir den Fehler nicht reproduzieren können, melden wir uns bei Ihnen, um weitere Details zu erfragen.
4. Fehler werden immer priorisiert behandelt und dann möglichst schnell geklärt.
5. Wenn es sich um einen Verbesserungsvorschlag handelt, wird der Vorschlag einem Milestone zugeordnet. Hier gibt es folgende Möglichkeiten:
   - <code>Not planned</code>: Für uns nicht interessant, weil es nicht in das
     Produkt passt.
   - <code>Backlog</code>: Interessant aber nicht mittelfristig geplant.
   - <code>vNext</code>: Interessantes Feature, dass wir mittelfristig
     entwickeln möchten.
   - <code>konkrete Versionsnummer</code>: Feature wird in einer der nächsten
     beiden Versionen umgesetzt.

:::note
Es kann passieren, dass sich unsere Prioritäten ändern und einzelnen geplante Features umpriorisiert werden.
:::
